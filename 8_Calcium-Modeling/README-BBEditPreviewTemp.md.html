Running Calcium Dynamics Simulations

Calcium simulations are run using the uG4 simulation framework, in this guide we will explain how to run the simulations by running a VRL-ug4 gui simulator, see VRL for downloading VRL studio. The VRL-ug4 gui will simulate the calcium dynamics inside a neuron, through activation by voltage dependent calcium channels (VDCC's), these channels follow the Borg-Graham model for calcium exchanges.

Software Requirements

  • VRL studio, this is the gui used for setting file paths and simulation parameters.
  • uG4 simulation framework
  • Optional: Gmsh, ParaView both for visualization of output data
  • Optional: ProMesh for visualizing the 3d geometry and converting from .swc to .ugx
  • Optional: To install ug4 manually (Windows, MacOS, Linux) please consult the instructions found here uG4-install, for windows it is recommended to install PowerShell for running terminal commands.

VRL-uG4 GUI Simulation (MacOS Catalina Version 10.15.5 & Windows 10)

The following steps have been tested on MacOS and Windows 10, the Linux version is still in development. To run a simulation using VRL:

  1. please install VRL studio, this is the gui used for setting file paths and simulation parameters.
  2. Then download the MacOS-VRL-CalciumDynamics folder to the Desktop of your computer. The folder MacOS-VRL-CalciumDynamics has the following contents

Below is a description of the folders and files:

  • geometry is the folder that contains the .swc geometry file
  • scripts is the folder containing the .lua script for ug4.
  • output this folder will contain the output data once the simulation is finished running
  • ug4 this is a static build of ug4, this is needed for the simulation to run, you will need to download a static build from here ug4-build, download the build that corresponds to your computer OS, and extract into the MacOS-VRL-CalciumDynamics folder.
  • iondynamics-01.vrlp this is the VRL studio project used for running the .lua script into ug4.
  • voltageData this is the voltage data that is used for the VDCC's,
  1. For a simulation to run you will need a geometry file in .swc format.
  2. Next move your voltage data into MacOS-VRL-CalciumDynamics/voltageData folder
  3. Double click the file iondynamics_01.vrlp to open the workflow in VRL studio. You will see the window below
  1. You will need to select the file locations for the uG4 folder, simulation script, Geometry .swc file, VM folder (voltage), and output folder. These are boxed in red in the figure above. Disregard the large code window. Set the other parameters to desired values, for now leave ER ON/OFF unchecked.
  • For the simulation script choose vdccFullCellCalcium_v2.lua inside the 'scripts' folder.
  • For the Geometry choose the file with .swc extension, make sure the geometry corresponds to the voltage data (used the output of previous step).
  1. Once you have selected the folders, files, and set the desired parameters, press the invoke button and the simulation will begin.
  2. You can monitor the progress of the simulation by clicking the bottom tab on the button and pulling the tab up.

  1. Once the simulation is completed the output will contain the following files and folders:

Below is a description of the files:

  • fullCalciumData.txt is a text file containing the calcium concentration at every node through time. Each row represents the calcium at one time step for all the nodes.
  • outDom.txt, outDom.swc, and outDom.ugx is the neuron geometry used given in different formats
  • timeSteps.txt is a text file containing the time at each time step.
  • run-simulation.sh is the bash shell script for running the code, for Windows 10 this is a PowerShell script and will have the .ps1 extension. Advanced users may use this as a template for running the simulation through the command prompt instead of the VRL gui.
  • meas is a folder containing the separate calcium concentrations for the different geometry subsets
  • vtk is a folder that contains the vtk output for the calcium concentrations and vtk output of the VDCC voltage data. Both files can be opened in Paraview.
